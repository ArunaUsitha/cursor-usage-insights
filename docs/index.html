Cursor Usage Dashboard

See exactly where your Cursor requests and tokens go — right inside Cursor. Cost dashboard, cache savings, model breakdown, and a cost simulator. No proxy, no login.

Install from Open VSX View on GitHub

Install

Published on the Open VSX Registry — the registry Cursor's Extensions view searches.

  1. Open the Extensions view in Cursor (Ctrl+Shift+X / Cmd+Shift+X).
  2. Search for "Cursor Usage Dashboard" and click Install.
  3. Run Cursor Usage: Open Dashboard from the command palette — that's it.

What's inside

Four tabs, from simple to detailed.

Status bar

A live usage figure that turns yellow at 80% and red at 95% of your quota, with a burn-rate projection.

Requests

The full request log with cost, cache savings, model filters, and CSV export.

Analyze

Rule-based findings — which model dominates your spend, cache efficiency, spike requests — each with a concrete fix.

Simulator

Replay any request's token profile against other models' published rates.

Privacy

Everything runs locally. The only network calls are to cursor.com — no proxy server, no separate login. The extension reads the session Cursor already created when you signed in. See the README for the full auth model.
